# Issue Log

## #1 Чат — пользователь видит чат с самим собой
- **Приоритет**: high
- **Статус**: fixed
- **Где**: `src/app/api/chat/find-or-create/route.ts:12`, `supabase/migration.sql`, `src/app/api/chat/route.ts:29`
- **Описание**: Учитель мог создать чат с собой (student_id = teacher_id). Существующие само-чаты остались в БД. Подозрение на утечку сообщений из само-чата ученику.
- **Решение**: (1) API теперь отклоняет `otherUserId === user.id`; (2) добавлена миграция SQL — DELETE само-чатов + CHECK constraint; (3) API фильтрует само-чаты из результатов; (4) проверка утечки: сообщения привязаны к chat_id, невозможно пересечение.
- **Исправлено**: 2026-07-06

## #2 Никнейм не синхронизирован между чатом и настройками
- **Приоритет**: high
- **Статус**: fixed
- **Где**: `src/app/(dashboard)/layout.tsx`
- **Описание**: Имя в хедере читалось из `user.user_metadata.full_name` (JWT), который мог быть устаревшим.
- **Решение**: `userName={profile?.full_name || user.user_metadata?.full_name || null}` — теперь сначала читаем из `profiles` (БД), потом из JWT.
- **Исправлено**: 2026-07-06

## #3 Настройки — учитель/админ видят роль «Ученик»
- **Приоритет**: high
- **Статус**: fixed
- **Где**: `src/app/(dashboard)/settings/page.tsx:14`, `src/app/(dashboard)/settings/settings-form.tsx:149`
- **Описание**: Две проблемы: (1) роль читалась из JWT, не из БД; (2) в settings-form проверялось только `role === "teacher"`, админ падал в else и показывал «Ученик».
- **Решение**: `page.tsx` — `profile?.role || user.user_metadata?.role`, `settings-form.tsx` — добавлена проверка `role === "admin"` → «Админ».
- **Исправлено**: 2026-07-06

## #3b Смена имени не сохраняется
- **Приоритет**: high
- **Статус**: fixed
- **Где**: `src/app/(dashboard)/settings/settings-form.tsx:35`, `src/app/api/fix-name/route.ts`
- **Описание**: `supabase.from("profiles").update()` через regular клиент блокировался RLS. Имя не сохранялось в БД.
- **Решение**: saveName теперь вызывает `POST /api/fix-name`, который через service client обновляет `profiles` и через admin client — `auth.users.user_metadata`.
- **Исправлено**: 2026-07-06

## #4 Зелёная точка — ученик с активной подпиской видит серый кружок / похожа на онлайн
- **Приоритет**: high
- **Статус**: fixed
- **Где**: `src/components/dashboard-header.tsx:214`
- **Описание**: Три проблемы: (1) `subscription_until` не читался из-за RLS; (2) серый кружок сбивал с толку; (3) зелёная точка ассоциировалась с «онлайн».
- **Решение**: (1) профиль теперь через `createServiceClient()`; (2) точка показывает ТОЛЬКО при активной подписке (нет серого варианта); (3) цвет изменён на оранжевый (🍊 бренд) с мини-🍊 внутри.
- **Исправлено**: 2026-07-06

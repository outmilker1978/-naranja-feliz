# Naranja Feliz — Release Notes

## v0.3.0 (2026-07-05) — Текущая

### Исправлено
- **Роли учителей**: все проверки роли теперь используют таблицу `profiles` (БД), а не `user_metadata` (JWT). Это чинит неотображение меню, редиректы, скрытые кнопки для учителей, назначенных до фикса.
- **Навигация**: панель учителя (Учителя, Проверка, Статистика) снова показывается в шапке для всех учителей.
- **Страница курсов**: учителя видят свои курсы в режиме «Учитель».
- **Страницы админки**: `/admin/courses/new`, `/admin/stats`, `/admin/courses/[id]`, `/admin/lessons/[id]` — добавлена проверка роли через `profiles`.
- **Оповещения**: ссылки из уведомлений ведут на правильные страницы (учитель → `/admin/submissions`, студент → `/courses/...`).

### Добавлено
- DragOrder (Составь предложение) — кликабельный вариант вместо drag-and-drop.
- ImagePick (Выбери изображение) — редактор с добавлением/удалением/загрузкой картинок.
- FillBlank (Вставка слова) — переписан на React JSX (работает автосохранение, проверка, подсветка).
- Choice (Выбор ответа) — поддержка старого формата (1-индексированный), нумерация 1) 2) 3).
- Image/Video — поддержка Google Drive (конвертация в прямую ссылку), Yandex Disk (внешняя карточка), VK Video, Rutube.
- `/api/upload-file` — универсальный endpoint загрузки файлов через service client (без RLS).
- `/api/submit-answer` — сохранение ответов + уведомление учителя с названием курса/урока.
- Уведомления: при ответе студента, комментарии студента, проверке учителя.
- Очистка ответов: кнопка «Очистить все ответы» с перезагрузкой страницы.

### Изменено
- SubmissionThread — убран `scrollIntoView` при опросе (оставлен только при отправке).
- Assign-teacher — обновляет `user_metadata` в дополнение к `profiles`.
- Review-submission — проверка роли через `profiles` (запасной вариант).
- Stats API — проверка роли через `profiles` (запасной вариант).

### Архитектурные решения
- Все загружаемые файлы → Supabase Storage (bucket `lesson-files`) через service client.
- Все интерактивные блоки должны использовать `/api/submit-answer` для создания уведомлений (кроме DragOrder и ImagePick — ещё не мигрированы).
- Роль учителя хранится в `profiles.role`; `user_metadata.role` — кэш, обновляется при назначении.

### Планы (ближайшие)
- Миграция DragOrder и ImagePick на `/api/submit-answer`.
- Видео-ответ (VideoAnswer).
- Личный чат студента с учителем.
- Дизайн главной страницы (портал школы).
- Монетизация.
- Деплой на Vercel.

## v0.2.0 (2026-07-01)

### Добавлено
- Блоки: текст, изображение, видео, fill_blank, choice, open_question, audio_answer, drag_order, image_pick.
- Админ-панель: управление курсами, уроками, блоками, учителями.
- Ролевая система: ученик / учитель (переключение через cookie).
- Система проверки: учитель просматривает ответы, оставляет комментарий, отмечает «проверено».
- Чат под каждым ответом (SubmissionThread) — студент и учитель могут переписываться.
- Уведомления: колокольчик с бейджем, страница `/notifications`.
- Прогресс: дольки апельсина на карточках курса и в списке уроков.
- Автозавершение урока при выполнении всех интерактивных блоков.
- Геймификация: уровни (Новичок → Профессор) в зависимости от числа пройденных уроков.
- Страница настроек: аватар, имя, email, пароль, роль.
- Supabase Storage: bucket `lesson-files`, загрузка аудио, изображений.
- API: `/api/upload-audio`, `/api/comments`, `/api/review-submission`, `/api/assign-teacher`, `/api/enroll`, `/api/notifications`, `/api/stats`.

## v0.4.4 (2026-07-12)

### Добавлено
- **Прокси-роут `/api/storage/[...path]`** — сервер-серверный fetch до Supabase Storage с Sharp-сжатием (1920px, JPEG q80). Решает ERR_CONNECTION_RESET при загрузке фото через провайдера.
- **Прокси для всех изображений** — все фото на главной, отзывах, о школе, контенте идут через `/api/storage/...` (Sharp-сжатие).
- **Серверная регистрация** — `POST /api/auth/signup` через `admin.createUser()` с auto-confirm (без письма). Создание профиля в `profiles`.
- **Серверный вход** — `POST /api/auth/login` через `signInWithPassword()` с cookie.
- **Сброс пароля** — `/forgot-password`, `/reset-password`, API `/api/auth/forgot-password`, `/api/auth/update-password`. Ссылка «Забыли пароль?» на странице входа.
- **Пакетное сжатие фото в Storage** — скрипт `scripts/compress-storage.mjs` прошёлся по всем bucket'ам, сжал >1MB файлы. Экономия: 5-10MB → 200-500KB.

### Исправлено
- **Auth callback** — использует `NEXT_PUBLIC_SITE_URL` вместо `request.url` (на Yandex Cloud request.url = http://0.0.0.0:8080).
- **Health check в деплое** — стучится на прямой URL контейнера, не на домен (SSL не сломан).
- **TypeScript** — `Buffer → Uint8Array` в proxy роуте (Next 16 не принимал Buffer как BodyInit).
- **Type error в сборке** — сброс `.next` кэша убрал ложную ошибку в сгенерированном validator.ts.

## v0.4.0 (2026-07-10)

### Добавлено
- **Портал школы** — главная страница собирается из секций (Hero, Features, About, Testimonials, FAQ, CTA) с сортировкой через `sort_order`. Блоки новостей, статей, рекламы — выезжающие карусели.
- **Админ-панель контента** (`/admin/content`) — иерархические блоки с перетаскиванием (▲▼). HERO и CTA заблокированы (первый/последний). Коллекции (Новости, Статьи, Реклама) — раскрывающиеся с внутренней сортировкой.
- **Фиксированная навигация на всех content-страницах** — панель прилипает под хедером при скролле:
  - `/content/[id]` — `← Все новости/статьи` + `Следующая новость/статья →`
  - `/courses/[courseId]` — `← Все курсы` + `Первый урок/Продолжить →`
  - `/courses/[courseId]/[lessonId]` — `← Название курса` + `Следующий урок →`
- **Сжатие изображений** при загрузке — Sharp (JPEG mozjpeg quality 82, PNG → WebP если >500KB, ресайз >1920px). При ошибке sharp — fallback на оригинал.
- **Signed URL (1 год)** для всех загружаемых файлов — гарантия доступа даже если bucket не публичный.
- **4-я карточка быстрого доступа** в Hero — «Проверка» (PenLine, для учителей) в дефолтах редактора.

### Исправлено
- **Загрузка фото**: если Sharp не может обработать файл — грузится оригинал (фото не пропадают).
- **Сохранение учителя**: PUT `/api/content/[id]` пропускает `category: null` (ошибка 500).
- **CTA sort_order**: при создании CTA-секции через POST присваивается `max + 1`.
- **Портал**: секции рендерятся в порядке `sort_order` (hero → ... → cta).
- **Страница отзывов**: текст ярко-белый (`text-white/95`) на тёмном фоне.
- **Удалён дубль `position: sticky`** из site-header (управляется только внешним wrapper'ом).

### Изменено
- **Sub-navigation**: `position: fixed; top: 73px` вместо sticky (на всех content-страницах).
- **Admin content list**: полный редизайн — группировка в блоки, locked HERO/CTA, expandable коллекции.
- **upload-file**: добавлена компрессия через Sharp с fallback при ошибке.
- **Hero editor**: заголовки разделов «Статистика» и «Карточки быстрого доступа» — жирный шрифт.

### Архитектурные решения
- Все изображения при загрузке проходят через Sharp (оптимизация размера) с try-catch fallback.
- Signed URL (1 год) — единый механизм доступа к файлам Storage.

## v0.2.0 (2026-07-01)

### Добавлено
- Блоки: текст, изображение, видео, fill_blank, choice, open_question, audio_answer, drag_order, image_pick.
- Админ-панель: управление курсами, уроками, блоками, учителями.

- Система проверки: учитель просматривает ответы, оставляет комментарий, отмечает «проверено».

- Уведомления: колокольчик с бейджем, страница `/notifications`.
- Прогресс: дольки апельсина на карточках курса и в списке уроков.
- Автозавершение урока при выполнении всех интерактивных блоков.
- Геймификация: уровни (Новичок → Профессор) в зависимости от числа пройденных уроков.
- Страница настроек: аватар, имя, email, пароль, роль.
- Supabase Storage: bucket `lesson-files`, загрузка аудио, изображений.
- API: `/api/upload-audio`, `/api/comments`, `/api/review-submission`, `/api/assign-teacher`, `/api/enroll`, `/api/notifications`, `/api/stats`.
